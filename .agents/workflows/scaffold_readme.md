# Scaffold Standard README Workflow

This workflow generates a high-quality `README.md` that acts as the front door for the project, following the Educational Ethics rule.

1. Create a `README.md` file in the project root.
2. Structure the file with the following sections:
   - **Project Title & Badges**
   - **Overview**: Explanation of the project's "Why Mandate".
   - **Architecture**: A brief note that the project adheres to the `AI_CONSTITUTION` (Modular First, 100-Line Law, `src/features/` structure).
   - **Prerequisites**: What tools need to be installed before running.
   - **Installation**: Step-by-step commands to clone and install dependencies.
   - **Running Locally**: Commands to start the dev server or application.
   - **Testing**: How to run the automated tests.
3. Review the generated content to ensure it is "Student Friendly" (explains complex setup simply).
